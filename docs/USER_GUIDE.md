# 使用说明

## 1. 软件简介

本软件是一个面向《三角洲行动》倒子弹流程的 Windows 桌面自动化辅助工具，主要围绕窗口绑定、图像模板识别、OCR 价格识别、自动点击和运行日志展开。

从随包文件和程序结构看，它包含以下能力：

- 辅助《三角洲行动》倒子弹相关购买、配装和出售流程
- 绑定目标窗口
- 识别指定界面或按钮图片
- OCR 读取价格、余额等数字信息
- 按配置流程执行购买、出售、收取等操作
- 记录运行状态和识别日志
- 支持定时、自动切换、智能价格分析等辅助功能

软件依赖同目录下的运行库、图片模板、字库和自动化组件。不要只单独复制 `jdgc721.exe` 运行。

## 2. 文件说明

主程序：

- `jdgc721.exe`：软件启动入口

关键依赖：

- `python312.dll`：Python 运行时
- `dm.dll`：大漠自动化组件
- `DmReg.dll`：大漠注册/调用组件
- `*.pyd`：Python 扩展模块
- `tcl`、`tk`、`ttkthemes`：界面运行资源

识别资源：

- `*.bmp`：界面按钮、状态、购买/出售结果等模板图
- `ziku.txt`、`ziku1.txt`：OCR 字库或特征库
- `newpeizhuang.txt`、`new余额.txt`、`房卡.txt`：识别规则或特征数据
- `temp_ocr_region.bmp`、`temp_ocr_region_processed.bmp`：OCR 截图与预处理临时图

## 3. 启动方式

### 视频教程

项目已随包附带操作视频：

- `operation_tutorial.mp4`

第一次使用建议先看视频，再按本文档操作。视频中演示了显示设置、游戏设置、版本选择、窗口绑定、参数填写和运行日志观察流程。

### 目录版

1. 打开本软件所在目录。
2. 双击 `jdgc721.exe` 启动。
3. 如果系统弹出安全提示，确认来源可信后再允许运行。
4. 首次启动可能需要等待几秒。

### 单文件版

如果使用已经打包好的单文件版本，例如 `306_single.exe`：

1. 双击 `306_single.exe`。
2. 程序会先自动解压内部文件，再启动真正的 `jdgc721.exe`。
3. 首次启动会比目录版更慢，这是正常现象。
4. 不要在程序运行中删除临时解压目录。

## 4. 使用前准备

启动前建议先完成这些检查：

1. 确认目标软件或游戏窗口已经打开。
2. 保持目标窗口分辨率、缩放比例和模板截图一致。
3. Windows 显示缩放建议使用 `100%`，否则图片识别可能偏移。
4. 不要删除同目录下的 `.bmp`、`.txt`、`.dll`、`.pyd` 文件。
5. 如果需要自动点击目标窗口，建议用管理员权限运行本软件和目标窗口，避免权限不一致导致点击失败。

### 视频教程中的推荐环境

教程里先调整了 Windows 显示设置，这是识别稳定性的关键步骤：

1. 打开 Windows `设置` -> `系统` -> `屏幕`。
2. 将缩放设置为 `100%`。
3. 将显示器分辨率设置为 `1920 x 1080`。
4. 将显示方向设置为 `横向`。
5. 保持 HDR、夜间模式等会改变画面颜色的功能关闭。
6. 关闭或避开会遮挡游戏画面的悬浮窗、弹窗、聊天窗口和录屏控制条。

如果你的显示器不是 1080p，也建议先按教程使用 `1920 x 1080 + 100% 缩放` 测试。这个软件依赖图片模板和固定区域 OCR，分辨率或缩放变化会导致按钮位置、价格区域和识别区域偏移。

### 版本选择

视频教程里展示了两个版本目录：

- `肉蛋（单独版本）`：只使用肉蛋相关功能时选择。
- `通用版本（金蛋肉蛋都在这里面下载）`：需要金蛋、肉蛋等多个功能时选择。

当前目录属于已经展开好的可运行目录。使用时不要只复制 `jdgc721.exe`，需要让主程序和同目录的模板图片、字库、DLL、Pyd 文件保持在一起。

### 安全软件检查

视频中打开过 Windows 安全中心的 `病毒和威胁防护` 页面。这个步骤主要用于确认软件文件是否被系统安全软件拦截或隔离。

如果软件打不开、文件缺失或运行后立即退出，可以检查 Windows 安全中心的保护历史记录。确认文件来源可信后，再恢复隔离文件或添加信任；不建议为了运行软件而长期关闭系统防护。

### 游戏页面准备

教程中目标窗口为 `三角洲行动`。建议按这个顺序准备：

1. 启动游戏并进入大厅。
2. 保持游戏窗口可见，不要最小化。
3. 按视频进入游戏 `设置` -> `视频` 页面检查画面设置，保持画面比例、分辨率和 UI 显示稳定。
4. 进入需要自动化处理的页面，例如大厅、配装、交易行、购买详情、兑换/出售页面。
5. 打开软件后，把软件窗口放在不遮挡关键按钮和价格区域的位置。
6. 正式开始前，先观察软件日志是否能识别到当前页面和价格。

## 5. 按视频教程操作流程

下面这套流程更接近视频教程中的实际演示：

1. 先按 `1920 x 1080`、`100%` 缩放、横向显示设置好 Windows 屏幕。
2. 进入 `三角洲行动`，确认游戏大厅、配装或交易行页面可以正常打开。
3. 如果是第一次使用，先按照教程确认游戏画面设置，避免窗口大小、画面风格或 UI 比例频繁变化。
4. 打开软件目录，运行 `jdgc721.exe` 或单文件版本。
5. 软件窗口标题通常显示为 `双流混合 - 客户端版`，右上角会显示当前状态，例如 `未运行`。
6. 点击 `购买窗口`，按日志提示把按钮拖到实际购买/交易页面所在的窗口后释放。
7. 点击 `检测窗口`，按日志提示把按钮拖到游戏主窗口后释放。
8. 在 `运行模式` 中按需要勾选 `启用降噪功能` 或 `启用自动出售功能`。
9. 在 `配装选择` 中选择需要使用的配装，例如 `上次配装`、`第一配装`、`第二配装`、`第三配装`、`第四配装`。
10. 根据当前物品和行情填写价格、延迟、数量等参数。
11. 点击 `开始` 或按 `F9` 开始运行。
12. 运行时盯住 `运行日志`，确认出现绑定成功、OCR 识别、子弹价格、购买或出售相关提示。
13. 需要暂停或停止时，再按 `F9` 开始/停止；如果需要立即强停，按 `End`。

## 6. 窗口绑定说明

软件需要知道要操作哪个窗口。视频中主要使用了两个绑定按钮：

- `购买窗口`：绑定实际购买或兑换窗口，用于识别价格、确认购买和后续操作。
- `检测窗口`：绑定主游戏窗口，用于识别大厅、配装、交易行等页面。

使用时建议：

1. 先把目标窗口打开并放到前台。
2. 点击 `检测窗口` 或 `购买窗口`。
3. 按日志提示，把绑定按钮拖到对应游戏窗口后释放。
4. 观察日志是否出现类似 `窗口绑定成功`、`返回值=1`、`窗口标题=三角洲行动` 的提示。
5. 两个窗口都绑定成功后，再设置参数并开始运行。

如果绑定失败：

- 确认目标窗口没有最小化。
- 确认软件和目标窗口权限一致。
- 尝试重新打开目标窗口。
- 尝试以管理员身份运行本软件。

## 7. 参数说明

不同版本界面文字可能略有差异，下面是常见参数含义。

`状态`

显示软件当前是否运行。常见状态包括 `未运行`、运行中或停止中。

`检测窗口`

用于绑定主游戏窗口。绑定成功后，软件才能根据模板图片判断当前处于大厅、配装、交易行或其他页面。

`购买窗口`

用于绑定购买、兑换或交易详情窗口。视频教程中会把这个窗口单独绑定，便于 OCR 读取价格并执行购买逻辑。

`开始`

开始执行自动化流程。界面同时提示 `F9:开始/停止`，所以也可以用 `F9` 快捷键控制。

`End:强停`

强制停止当前流程。遇到识别异常、页面跳错、鼠标持续点击等情况时优先使用。

`隐藏窗`

用于隐藏软件窗口，减少遮挡。但第一次调试时不建议隐藏，先看日志更容易排查问题。

`启用降噪功能`

用于提高截图或 OCR 识别稳定性。识别价格不稳、日志中价格跳动明显时可以尝试开启。

`启用自动出售功能`

用于在购买或配装流程后自动进入出售相关步骤。未确认价格和页面前，不建议第一次就开启。

`配装选择`

用于选择使用哪套配装流程。视频中能看到 `上次配装`、`第一配装`、`第二配装`、`第三配装`、`第四配装` 等选项。选择不同配装会影响软件识别和处理的物品数量、页面位置或后续步骤。

`购买延迟`

控制执行购买动作前后的等待时间，单位通常为毫秒。数值过小可能导致界面还没刷新就继续点击。

`识别延迟`

控制 OCR 或图片识别的间隔。数值越小反应越快，但也更容易误判或增加 CPU 占用。

`子弹价格 MIN / MAX`

用于价格识别或筛选区间，软件可能会根据区间判断是否继续购买或出售。

`挡尿价格上限`

视频中该项用于设置价格上限。高于上限时，软件应避免按购买逻辑继续执行。

`配装1子弹数量`

第一套配装中的子弹数量。视频示例中会根据这个数量判断购买或补充逻辑。

`最低价格`

用于出售时的最低价格限制，避免出售价格低于预期。视频中该项位于 `出售差价` 旁边，需要结合当前行情填写。

`出售差价`

用于控制出售阶段与当前价格之间的差值。需要结合实际行情谨慎设置。

`定时启动 / 定时关闭`

用于在指定时间自动开始或停止。

`当前时间`

显示软件读取到的当前系统时间，主要用于核对定时启动和定时关闭是否设置正确。

## 8. 视频教程中的页面要点

教程里出现了几个关键游戏页面，使用时尽量保持页面与教程一致：

- `大厅页面`：用于启动前检测窗口和确认游戏已进入正常状态。
- `配装页面`：用于选择方案、查看背包、弹药和装备价格。
- `交易行/购买页面`：用于识别物品价格、数量和购买按钮。
- `兑换/详情页面`：用于识别平均单价、数量和兑换按钮。
- `行动地图页面`：视频中出现了长弓溪谷等页面，说明软件可能会在流程中识别大厅或行动入口状态。

如果软件日志提示没有识别到页面，先不要反复点击开始。应先回到教程中相同的页面，再重新绑定窗口并测试。

## 9. 运行日志

运行日志是判断软件是否正常工作的关键位置。

重点观察：

- 是否成功绑定窗口
- 是否识别到当前界面
- OCR 是否识别出数字
- 是否出现购买失败、出售失败、未命中模板等提示
- 是否触发自动停止或线程退出

视频中能看到的典型日志包括：

- `请拖动按钮到购买窗口后释放`
- `副窗口绑定成功`
- `请拖动按钮到检测窗口后释放`
- `检测窗口绑定成功`
- `子弹价格`
- `通过F9快捷键停止程序`
- `程序已停止`
- `OCR识别线程退出`

如果日志一直没有变化，通常说明程序没有进入实际运行流程，或目标窗口没有绑定成功。

## 10. 常见问题

### 双击后没有界面或很快退出

可能原因：

- 文件不完整
- 杀毒软件拦截
- 缺少运行库或权限不足
- 目标环境不符合程序预期

处理建议：

- 确认整个目录完整存在。
- 不要只运行单独的 `jdgc721.exe`。
- 尝试以管理员身份运行。
- 查看是否有安全软件隔离了 `dm.dll`、`DmReg.dll` 或 `.pyd` 文件。
- 如果安全软件拦截，先确认文件来源可信，再添加信任或恢复隔离文件；不建议不加判断地长期关闭系统防护。

### OCR 识别不准

可能原因：

- 分辨率或缩放比例变化
- 模板图与当前界面不匹配
- 目标区域被遮挡
- 字库或临时 OCR 文件缺失

处理建议：

- 将 Windows 显示缩放调到 `100%`。
- 将显示分辨率调到教程中的 `1920 x 1080`。
- 保持目标窗口大小和位置稳定。
- 不要遮挡目标窗口。
- 确认 `ziku.txt`、`ziku1.txt` 和 OCR 临时图文件存在。

### 点击没有效果

可能原因：

- 目标窗口没有激活
- 软件权限低于目标窗口
- 坐标偏移
- 大漠组件未正常加载

处理建议：

- 重新绑定窗口。
- 以管理员身份运行。
- 确认 `dm.dll` 和 `DmReg.dll` 没有被删除或拦截。
- 检查目标窗口是否被其他窗口遮挡。

### 单文件版启动慢

单文件版需要先解压内部资源再启动，所以第一次启动较慢。目录版通常启动更快。

### 开始后页面跳错或一直点错

可能原因：

- 当前页面和教程页面不一致
- 游戏窗口缩放或位置变化
- 绑定了错误窗口
- 参数设置过激进，例如延迟太低

处理建议：

- 按 `End` 强停。
- 回到大厅或配装页面。
- 重新点击 `购买窗口` 和 `检测窗口` 绑定。
- 把 `购买延迟`、`识别延迟` 适当调大。
- 从不勾选自动出售开始测试，确认购买识别稳定后再开启更多功能。

## 11. 注意事项

- 本软件会模拟窗口识别、OCR 和点击操作，请只在你有权限操作的环境中使用。
- 不建议在重要账号或不可恢复场景中直接使用未经验证的自动化流程。
- 修改 `.bmp` 模板、字库或规则文件前建议先备份。
- 打包成单文件后，运行时仍会在临时目录展开完整资源，这是正常现象。
- 如果要分发给他人，请同时说明软件用途、风险和适用环境。
- 如果后续准备开源到 GitHub，建议把使用风险、依赖组件来源、适用分辨率和免责声明写在 README 的显眼位置。

## 12. 快速排查清单

遇到问题时可以按顺序检查：

1. 文件是否完整。
2. 是否以管理员身份运行。
3. 目标窗口是否打开并可见。
4. 是否成功绑定目标窗口。
5. Windows 显示缩放是否为 `100%`。
6. 显示分辨率是否为 `1920 x 1080`。
7. 当前游戏页面是否和教程中的页面一致。
8. 是否误开了会遮挡价格、按钮或背包区域的窗口。
9. 日志是否出现 OCR、模板识别或点击错误。
10. 安全软件是否拦截 DLL 或 EXE。
11. 单文件版是否有足够时间完成解压启动。
